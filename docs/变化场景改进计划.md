# 大变化场景下的时空融合改进方案与实施计划

## 1. 问题定义

当前模型在变化较大样本上的性能下降，核心症状是：
- 预测结果过度依赖 `fine_t1` 的纹理与结构
- 对 `coarse_t2` 的变化信息利用不足
- 在地物变化区域出现“复制旧时相细节”的现象

结论：模型存在条件捷径学习（shortcut），优化过程倾向使用更容易拟合的 `fine_t1`，而不是学习 `t2` 的真实变化。

## 2. 改进目标

1. 提升大变化样本上的重建质量
2. 降低对 `fine_t1` 的过度依赖
3. 强化 `coarse_t2` 对最终预测的控制力
4. 尽量保持最小改动和高复用，不破坏现有 CLI/配置/训练流程

## 3. 总体策略（按优先级）

### P1（最小侵入，优先做）
1. `fine_t1` 条件 dropout / masking（训练时）
2. 变化区域加权损失（change-aware weighting）
3. 粗分辨率一致性损失（prediction downsample 后约束到 `coarse_t2`）

### P2（中等改动）
1. 残差建模：预测 `delta = fine_t2 - fine_t1`
2. 双分支融合（静态纹理分支 + 变化分支）
3. change gate 动态融合 `fine_t1` 与 `coarse_t2` 信息

### P3（中高改动）
1. 变化强度 curriculum（先低变化后高变化，或分桶采样）
2. 大变化样本 hard mining
3. 蒸馏/少步采样优化（扩散分支推理效率）

## 4. 先做验证（不改主干前）

## 4.1 诊断实验
1. 条件消融
- A: 正常输入
- B: 去掉 `fine_t1`
- C: 去掉 `coarse_t2`
- 观察大变化样本性能降幅，量化依赖关系

2. 分桶评估
- 按变化强度将样本分为 low / mid / high
- 分别统计 RMSE/MAE/SSIM/SAM/CC

3. 复制倾向指标
- 统计 prediction 与 `fine_t1` 的相似度，观察其在 high-change 桶是否异常高

## 4.2 验收门槛（第一阶段）
- high-change 桶 RMSE 或 MAE 至少改善 5%
- overall 指标不回退超过 1%

## 5. 第一阶段实施计划（建议 1-2 周）

### Step 1: 仅加 `fine_t1` dropout（最低风险）
- 在模型前向中对 `fine_t1` 做随机 dropout/masking
- 默认关闭（`p=0.0`），通过配置开关启用

预期：缓解模型对 `fine_t1` 的绝对依赖

### Step 2: 增加变化区域加权损失
- 用 `|coarse_t2 - coarse_t1|` 估计变化图（可平滑）
- 在变化区域提高重建误差权重

预期：模型对变化区域学习更积极

### Step 3: 增加 coarse 一致性损失
- 将预测下采样到 coarse 分辨率
- 与 `coarse_t2` 对齐（L1/L2）

预期：增强时相一致性，降低旧时相复制

### Step 4: 网格搜索（小范围）
- `dropout_p`: 0.05 / 0.1 / 0.2
- `change_weight`: 1.0 / 2.0 / 4.0
- `coarse_consistency_weight`: 0.1 / 0.5 / 1.0

输出最佳组合与 trade-off 结论

### Step 5: 高频约束最小实验矩阵（Flow）

在保持当前 baseline 流程不变的前提下，只通过配置增量开启高频损失：

1. 组1（baseline）
- 配置：`configs/flow/change_aware_toy.py`
- 目的：提供统一对照

2. 组2（+L_grad）
- 配置：`configs/flow/change_aware_toy_hf_grad.py`
- 核心：变化区域梯度约束（`grad_loss_weight > 0`）
- 验证：确认高频边缘监督是否有效

3. 组3（+L_grad+L_lap）
- 配置：`configs/flow/change_aware_toy_hf_grad_lap.py`
- 核心：叠加多尺度 Laplacian 约束
- 验证：判断是否需要比一阶梯度更强的多尺度高频监督

4. 组4（+L_grad+L_lap+ranking）
- 配置：`configs/flow/change_aware_toy_hf_grad_lap_rank.py`
- 核心：增加“更接近 target、远离 reference”的排序约束
- 验证：直接检验“复制 fine_t1”偏置能否被抑制

默认阈值策略：
- `hf_mask_strategy='quantile'`
- `hf_mask_quantile=0.8`

建议判据：
- `TRP` 是否从负值上移（越接近正值越好）
- 变化区域视觉细节是否更接近 `fine_t2`
- 全局指标（RMSE/MAE/SSIM）不出现明显回退

## 6. 代码改动建议（最小化路径）

优先原则：先改配置和模型适配层，不改引擎主循环。

### 6.1 首选改动点
1. `stf/models/`（主改动）
- 增加条件 dropout
- 增加残差输出或 change gate（后续）

2. `configs/*.py`
- 增加可配置超参：
  - `condition_dropout_p`
  - `change_loss_weight`
  - `coarse_consistency_weight`

3. `stf/engine/train.py`（必要时最小改动）
- 仅在现有 loss 汇总处加入额外 loss 项
- 不修改数据流与 CLI 接口

### 6.2 尽量不改
- `stf/cli/*`
- `stf/config/loader.py`
- `stf/compat/*`

## 7. 风险与规避

1. 风险：dropout 过强导致整体细节退化
- 规避：从小值开始，配合 high-change 指标选型

2. 风险：变化权重过大导致噪声放大
- 规避：变化图平滑 + 权重上限

3. 风险：新增 loss 破坏训练稳定性
- 规避：warmup 阶段仅主损失，后续逐步加权
